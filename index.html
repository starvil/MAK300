Stage 02 参数估计改进 — 引擎A：LSTM + GNN 双分支协同
L
LSTM 分支 → 预期收益率 μ
基于多源时序特征，通过门控机制捕捉长期依赖与短期均值回复性，替代传统样本均值估计。
01双层 LSTM（hidden_size=128）提取时序特征
02Attention Pooling 加权聚合多时间步信息，非仅取最后一步
03股票嵌入向量 ei（dim=16）拼接，捕捉个股异质性
04全连接层（64→32→1）输出 μi,t+1
LLSTM = Σ(ri,t+1 − μi,t+1)² + λ₁‖θLSTM‖² + λ₂‖ei‖²
OUTPUT: μ ∈ ℝ300
G
GNN 分支 → 协方差矩阵 Σ
构建多层图结构（行业边、历史相关性边、指数共现边），利用图注意力网络聚合邻居信息，输出低秩协方差矩阵。
01三层图结构：行业边(ω=1.0) + 相关性边(ρ>0.6) + 共现边(ω=0.3)
02双层 GAT（heads=8→4，output=64→32）自适应学习邻居重要性
03低秩+对角分解：Σ = L·LT + diag(σ²)，L∈ℝN×k，k≪N
04均值修正头输出 Δμ，与 LSTM 预测相加得到最终预期收益率
LGNN = ½log|Σ| + ½(r−μ)TΣ−1(r−μ) + λlap·Tr(FTLF)
OUTPUT: Σ ∈ ℝ300×300
μ (LSTM)
联合微调 · 最大化组合夏普比率
Ljoint = −Sharpe(ω*) = − (μpTω*) / √(ω*TΣω*)
Σ (GNN)
LSTM 与 GNN 不再"各自为政"，朝组合优化最优方向协同进化
▼
ω*
均值-方差优化求解 → 目标权重 ω*
将 LSTM 预测的 μ 和 GNN 预测的 Σ 代入带约束的均值-方差二次规划，输出最优资产权重。
◆ 优化模型
  • min ½ωTΣω（最小化组合方差）
  • ωTμ = μ₀（目标收益约束）
  • ωT1 = 1（满额投资）
  • ωi ≥ 0（不允许卖空 → QP数值求解）
    • ◆ 风控约束
  • 跟踪误差（与沪深300权重欧氏距离）≤ 5%
  • 单票最大权重 ≤ 3%（硬性风控）
  • 输出目标权重 ω* ∈ ℝ300
  • ω* 传递给引擎B作为"指导信号"

    Stage 03 组合优化与动态执行 — 引擎B：深度强化学习
    B
    引擎B：PPO 深度强化学习动态再平衡
    将再平衡问题建模为马尔可夫决策过程（MDP），采用 PPO 算法的 Actor-Critic 架构，每日输出连续动作空间下的调仓决策。
    State 状态空间
  • 当前持仓权重 ωt
  • 目标权重 ω*t（引擎A输出）
  • 偏离程度 ωt − ω*t
  • 现金比例
  • 市场特征（沪深300动量、波动率、成交量比）
  • 行业轮动指标 / 波动率状态
  • 预估冲击成本与买卖价差
  • 距下次定期调仓日天数
    • Action 动作空间
  • 连续空间 αi ∈ [-1, +1]300
  • +1 调至最大允许权重
  • -1 清仓该股票
  • 0 维持当前持仓
  • 映射: ωt+1,i = ωt,i + η·αi·(ω*i − ωi)
  • η=0.3：单日最多调整偏离量30%
  • 掩码: 偏离<0.5%的维度强制为0
    • Reward 奖励函数
    Rt = rportfolio
    − γ₁ · ctrading
    − γ₂ · TEt
    − γ₃ · DDt
    + γ₄ · αexcess
  • rportfolio — 组合收益
  • ctrading — 交易成本（含滑点与冲击）
  • TEt — 跟踪误差
  • DDt — 回撤
  • αexcess — 超额收益
  • γ 权重根据波动率状态自适应调整（详见下方）
    • Transition 状态转移
  • St+1 = f(St, αt, Markett+1)
  • 由真实市场数据驱动
  • 无需显式建模转移概率
  • 回测环境中获取交互数据
    • ⚖
    自适应权重机制 · 根据市场波动率状态动态调整 γ
    定义波动率状态因子：βt = σt / σbase − 1，其中 σt = √252 · std(r300,t-19 … r300,t)，σbase 为过去252日年化波动率中位数。 基础权重 γbase = (γ₁=0.05, γ₂=0.1, γ₃=0.05, γ₄=0.02)。
    ▲ 高波动状态（βt > 0.2）→ 转向防守
    γ₁ = γ₁base × (1 + 0.5βt) → 交易成本惩罚增大，减少交易
    γ₂ = γ₂base × (1 + βt) → 跟踪误差惩罚增大
    γ₃ = γ₃base（不变） → 回撤惩罚保持
    γ₄ = γ₄base × 0.5 → 超额奖励降低，避免过度冒险
    VS
    ▼ 低波动状态（βt < -0.1）→ 转向进取
    γ₁ = γ₁base × 0.8 → 交易成本惩罚降低，可频繁调仓
    γ₂ = γ₂base × 0.7 → 跟踪误差惩罚降低，允许适度偏离
    γ₃ = γ₃base（不变） → 回撤惩罚保持
    γ₄ = γ₄base × (1 − βt) → 超额奖励增大，鼓励进取
    π
    PPO Actor-Critic 架构 · 双网络并行决策
    共用输入
    状态 St ∈ ℝdstate（持仓 + 目标权重 + 市场状态 + 成本估计 + 时间信息）
    Actor · 策略网络 π(a|s)
    输出交易动作 at
    接收状态 St，输出300维连续动作的高斯分布参数
    FC(512) → ReLU → FC(256) → ReLU → FC(128) → ReLU
    ├─→ 均值头 μa ∈ ℝ300（线性输出）
    └─→ 标准差头 σa ∈ ℝ300（Softplus激活，保证正值）
    at ~ N(μa, σa²) · 训练时加探索噪声
    Critic · 价值网络 V(s)
    评估状态价值 V(St)
    接收状态 St，输出标量状态价值估计，用于计算优势函数
    FC(512) → ReLU → FC(256) → ReLU → FC(128) → ReLU
    └─→ V(St) ∈ 标量（线性输出）
    GAE: λ=0.95, γ=0.99 · Clipped Surrogate Objective
    Actor 与 Critic 共享输入层，各自由独立的三层全连接网络分支处理； Actor 输出交易动作，Critic 评估该动作对应的长期价值，两者协同指导策略更新

    Alpha 增强组合持仓 · 动态最优执行路径
    引擎A 回答「组合应该是什么样」（配置方向） · 引擎B 回答「什么时候调、调多少」（执行策略）
    Defense 多层次风控体系
    模型风险
    LSTM / GNN 定期重训练；RL策略设为混合策略（RL×80% + 引擎A×20%），避免策略突变
    过拟合风险
    Walk-forward 交叉验证；LSTM 使用 Dropout + L2 正则化；GNN 图结构引入先验知识减少数据依赖
    极端市场风险
    硬性风控：单日最大调仓 ≤ 10%、单股最大权重 ≤ 3%，作为 RL 动作的裁剪边界
    Alpha衰减风险
    滚动窗口评估 Alpha 时变性；连续 N 期为负则触发降权或切换至纯基准跟踪
    Summary 理论创新点总结
    Σ
    策略核心理论创新点
    创新维度 具体贡献与改进说明
    输入参数敏感性改进 用 LSTM 替代样本均值估计预期收益率 μ，融合量价、基本面、资金流、市场状态等多源特征；用 GNN 替代样本协方差矩阵估计 Σ，通过低秩+对角分解将参数量从 O(N²) 降至 O(Nk)。两者通过最大化组合夏普比率的联合微调实现协同优化，从根源上缓解了马科维兹模型最致命的输入参数敏感性与估计误差放大问题。
    收益率分布假设放松 GNN 的低秩 + 对角分解结构不依赖正态分布假设；高斯似然损失函数能够隐式学习肥尾特征——当真实收益率出现极端下跌时，损失函数会强烈惩罚，迫使模型学习尾部风险。GNN 的图注意力机制还引入了非线性激活，超越了传统线性因子模型的表达能力。
    静态模型向动态决策的突破 引擎B的 RL 框架将单期优化扩展为多期动态决策，将交易成本、市场冲击、跟踪误差等复杂因素统一纳入奖励函数进行端到端优化。自适应权重机制使策略在高波动期自动转向防守（增大跟踪误差惩罚、降低超额奖励）、低波动期转向进取（降低交易成本惩罚、增大超额奖励），实现了对不同市场状态的动态响应。
    沪深300指数增强的应用创新 LSTM + GNN 的输出直接作为 Alpha 信号来源；改进后的均值-方差优化叠加跟踪误差约束确保策略是「增强」而非「偏离」基准；RL 的动态再平衡优化了执行层面的 Alpha 损耗（大单冲击成本 vs 延迟追踪风险的权衡），形成从信号生成到交易执行的完整闭环。
